Documentation

Add prompt versioning, test cases, and AI-judged evals to your agents.

#Installation

Install biscotti with pip or uv. Then follow the Quick Start to wire it into your app.

terminal 
pip install biscotti

With uv (recommended):

terminal 
uv add biscotti

Try the demo without installing:

terminal 
uvx biscotti dev

Opens a demo playground at http://localhost:8000/biscotti with sample agents pre-configured.

#Quick Start

Three steps to add biscotti to any FastAPI app. For more mounting options, see the Advanced section.

Step 1: Decorate your agent

app.py 
from biscotti import Biscotti, biscotti

@biscotti(
    name="recipe agent",
    description="Suggests recipes based on ingredients",
    default_system_prompt="""You are a creative chef.
Ingredients: {{ingredients}}
Occasion: {{occasion}}""",
)
async def recipe_agent(user_message: str, system_prompt: str) -> str:
    # your LLM call here
    ...

The decorator registers your agent, auto-detects {{variable}} placeholders (see Variables), and seeds the first prompt version on startup.

Step 2: Mount in FastAPI

app.py 
from fastapi import FastAPI

app = FastAPI()
bi = Biscotti()
app.mount("/biscotti", bi.app)

Step 3: Open the UI

Navigate to http://localhost:8000/biscotti and share the URL with your team. Next: write test cases and run your first eval.

#Your First Eval

  1. Select an agent in the sidebar
  2. Add test cases in the Test Cases tab -- give each a name and user message
  3. Open the Evals tab and click "Generate from Prompt" to auto-create scoring criteria
  4. Set your judge model -- this requires an API key (environment variable or the Settings panel)
  5. Click "Run Eval" to score all test cases against your criteria
  6. Edit the system prompt, save a new version, and re-run the eval to compare scores

For details on how scoring works, see the Eval System. After running an eval, use Coaching to get AI-generated improvement suggestions.

#Prompt Versioning

On first startup, biscotti seeds v1 from your default_system_prompt (set during Quick Start) and promotes it to live.

The workflow:

  1. Edit the prompt in the UI and save as v2, v3, etc. -- new versions always start as drafts
  2. Promote a draft to live -- this archives the previous live version
  3. Your agent automatically uses the live version at runtime -- no restart, no redeploy

Each version tracks the full prompt text, a status (draft / current / archived), optional notes, and a timestamp. Run evals against different versions to compare performance before promoting.

#Test Cases

Test cases are named inputs you run against your agent. Each test case has:

  • Name -- a descriptive label (e.g., "simple greeting", "edge case - empty input")
  • User message -- the input text sent to the agent
  • Variable values -- optional key-value pairs for {{variable}} placeholders (see Variables)

Create test cases in the UI or via the REST API. They persist across sessions and are used by the batch eval system.

#Variables

Use {{variable}} placeholders in your system prompts to make them dynamic. biscotti auto-detects them from the prompt text.

app.py 
@biscotti(
    name="support agent",
    default_system_prompt="""You are a {{tone}} support agent for {{product}}.
Always respond in {{language}}.""",
)
async def support_agent(user_message: str, system_prompt: str) -> str:
    # system_prompt arrives with variables already substituted
    ...

Set variable values per test case or per run. The UI shows input fields for each detected variable. You can also pass values programmatically via the extended callable params dict.

#Eval System

biscotti includes an LLM-as-judge evaluation system. You need an API key configured for at least one provider before running evals.

  1. Generate criteria -- click "Generate from Prompt" to auto-create scoring rubrics from your system prompt
  2. Configure judge model -- pick any connected model (Anthropic, OpenAI)
  3. Run batch evals -- score all test cases against your criteria in one click
  4. Track history -- compare scores across prompt versions

Each criterion has a name, description, and 1-5 scoring scale. The judge model evaluates each test case output against each criterion and returns a score with reasoning. After an eval completes, use Coaching to get targeted improvement suggestions.

#Coaching

After running an eval, click "Get Coaching" to get AI-generated suggestions for improving your system prompt. The coach analyzes:

  • Which criteria scored low and why
  • Patterns across test cases
  • Specific prompt edits to improve scores

Configure the coach model separately from the judge model in Settings. Both require an API key.

#PydanticAI

Wrap a PydanticAI agent with the @biscotti decorator. For richer telemetry, return a dict instead of a string. To accept UI-configured overrides, add the extended callable signature.

app.py 
from pydantic_ai import Agent
from biscotti import biscotti

pydantic_agent = Agent(model="openai:gpt-4o")

@biscotti(name="recipe agent")
async def recipe_agent(user_message: str, system_prompt: str) -> str:
    result = await pydantic_agent.run(
        user_message,
        system_prompt=system_prompt,
    )
    return result.output

#Anthropic SDK

Use the Anthropic Python SDK directly. See Return Types for dict-based telemetry and Extended Callable for accepting UI overrides.

app.py 
import anthropic
from biscotti import biscotti

client = anthropic.AsyncAnthropic()

@biscotti(name="chat agent", default_system_prompt="You are helpful.")
async def chat_agent(user_message: str, system_prompt: str) -> str:
    response = await client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        system=system_prompt,
        messages=[{"role": "user", "content": user_message}],
    )
    return response.content[0].text

#OpenAI SDK

Use the OpenAI Python SDK directly. See Return Types for dict-based telemetry and Extended Callable for accepting UI overrides.

app.py 
from openai import AsyncOpenAI
from biscotti import biscotti

client = AsyncOpenAI()

@biscotti(name="gpt agent")
async def gpt_agent(user_message: str, system_prompt: str) -> str:
    response = await client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_message},
        ],
    )
    return response.choices[0].message.content

#Extended Callable

Your callable can accept an optional third params dict to receive UI-configured overrides:

app.py 
@biscotti(name="my agent", default_system_prompt="...")
async def my_agent(user_message: str, system_prompt: str, params: dict) -> str:
    model = params.get("model", "claude-sonnet-4-6")
    temperature = params.get("temperature", 1.0)
    # Use model and temperature in your SDK call...

Available params: model, temperature, reasoning_effort, variable_values. Combine this with dict return types for full telemetry.

#Return Types

Your decorated function can return a string or a dict. Return a string for simple responses:

python 
async def my_agent(user_message: str, system_prompt: str) -> str:
    return "the response text"

Return a dict to capture token usage and model metadata:

python 
async def my_agent(user_message: str, system_prompt: str) -> dict:
    return {
        "output": "the response text",
        "input_tokens": 150,
        "output_tokens": 80,
        "model": "gpt-4o",
    }

When you return a dict, biscotti tracks token usage and model info in the run history. Use this with the extended callable signature for full control over model selection and telemetry.

#Data Models

Key models used by the REST API:

AgentMeta

Agent registration info (name, description, variables, tags, models).

PromptVersion

A versioned prompt with fields: id, agent_name, version_number, body, status (draft/current/archived), notes, created_at.

TestCase

Test input with fields: name, user_message, variable_values.

RunLog

Execution record with fields: id, agent_name, version_id, user_message, output, input_tokens, output_tokens, latency_ms, model, cost, created_at.

EvalRun

Batch eval with fields: id, agent_name, version_id, judge_model, criteria, results (per test case per criterion), aggregate_scores, created_at.

#API Keys

The eval and coaching systems need API keys for their LLM judge and coach models.

Resolution order: environment variable, then in-memory UI key, then none.

Environment variables

terminal 
export ANTHROPIC_API_KEY=sk-ant-...
export OPENAI_API_KEY=sk-...

UI settings panel

Open Settings in the main app and enter keys per provider. Keys are session-only and never written to disk.

#Storage Options

python 
# SQLite file (default)
Biscotti(storage="biscotti.db")

# In-memory (for tests)
Biscotti(storage=":memory:")

# Absolute path
Biscotti(storage="sqlite:////var/data/biscotti.db")

SQLite is the only storage backend. Data persists across restarts, and the database is auto-initialized on first startup. Pass the storage option when configuring your Biscotti instance.

#Export/Import

Export an agent's full configuration (prompt versions, test cases, settings) as JSON via the REST API:

http 
GET /biscotti/api/agents/{name}/export

Import into another instance:

http 
POST /biscotti/api/agents/{name}/import

Use this to share agent configurations across environments or back up before major changes.

#Mounting Options

Choose how biscotti integrates with your FastAPI app. Combine with Storage Options to control where data lives.

As a sub-app (recommended)

python 
app = FastAPI()
bi = Biscotti()
app.mount("/biscotti", bi.app)

As an included router

python 
app = FastAPI()
bi = Biscotti()
app.include_router(bi.router, prefix="/biscotti")

Custom title

python 
bi = Biscotti(title="My Prompt Studio")

Context manager for clean shutdown

python 
async with Biscotti() as bi:
    app.mount("/biscotti", bi.app)
